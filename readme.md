# 30 Days of Cloud DevOps and Automation

This repository documents my hands-on DevOps learning journey. I will be building practical automation skills by working through Linux, scripting, containerisation, infrastructure as code, CI/CD, and cloud deployment.

The focus is on doing, not theory. Each phase ends with a working system I can run and demonstrate.

---

## What I will build

By the end of these 30 days, I will have:

- A Bash-based automation script that deploys a Dockerised application to a remote Ubuntu server
- Cloud infrastructure provisioned using Terraform
- Server configuration automated with Ansible
- A CI/CD pipeline using GitHub Actions
- A containerised application deployed to a cloud VM
- Basic monitoring using Prometheus and Grafana
- An end-to-end DevOps workflow that is repeatable and fully automated

---

## Tech Stack

- Linux (Ubuntu)
- Bash scripting
- Docker and Docker Compose
- Git and GitHub
- Terraform
- Ansible

## Phase 1: Linux, Bash, and Docker Automation

In this phase, I will focus on building strong fundamentals in Linux, scripting, and containerisation.

I will work on:

- Linux system operations, permissions, and process management
- Bash scripting for automation tasks
- Docker image creation and container management
- Docker Compose for multi-container setups
- SSH-based automation for remote execution

Deliverable:
- A Bash script that installs dependencies and deploys a Dockerised application to a remote Ubuntu server

---

## Phase 2: Infrastructure as Code (Terraform and Ansible)

In this phase, I will move from manual setup to reproducible infrastructure.

I will work on:

- Provisioning cloud infrastructure using Terraform
- Structuring Terraform code into reusable modules
- Managing remote state
- Writing Ansible playbooks for server configuration
- Combining Terraform and Ansible into a single workflow

Deliverable:
- A fully automated infrastructure setup that provisions and configures a server in one run

---

## Phase 3: CI/CD with GitHub Actions

In this phase, I will automate the build and deployment process.

I will work on:

- Building CI pipelines using GitHub Actions
- Running tests and building Docker images on push
- Pushing images to a container registry
- Automating deployment to a cloud VM
- Managing secrets and environment variables
- Adding health checks to deployments

Deliverable:
- A CI/CD pipeline that automatically builds and deploys my application on every push

---

## Phase 4: Cloud-Native Deployment and Observability

In this phase, I will focus on resilience, scaling, and monitoring.

I will work on:

- Kubernetes basics for deployment
- Blue-green deployment strategy using Nginx or Kubernetes
- Setting up Prometheus and Grafana for monitoring
- Exploring autoscaling concepts
- Applying basic security hardening practices

Deliverable:
- A full cloud-native system with deployment automation, monitoring, and controlled release strategy

---

## Final Outcome

By the end of this journey, I will have built a complete DevOps workflow that includes:

- Infrastructure provisioning
- Configuration automation
- Continuous integration and deployment
- Containerised application delivery
- Monitoring and observability
- Repeatable cloud deployment from scratch
